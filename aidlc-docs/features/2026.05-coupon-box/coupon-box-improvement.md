## 2. 문제 정의 & 솔루션 요약

**문제**: 쿠폰함 진입 후 다음 행동(CTA)이 상세페이지 뒤에 숨어있어 불필요한 클릭 발생. 쿠폰 1개뿐인 상황에서도 35.84% 이탈.

**솔루션**: 상세페이지 제거, 목록에 하단 고정 CTA 삽입, 최대 혜택 쿠폰 자동 선택으로 즉시 행동 시작 가능하게 함.

**대상 사용자**:
- 핵심: 핀다에서 대출 조회 이력 있으나 미약정 사용자
- 부가: 타사에서 대출 조회 이력 있으나 미약정 사용자

## 5. 서비스 프로세스 (User Flow)

```

  │
  ├─ clickYn=true인 사용 가능 쿠폰 있음?
  │     │
  │     ├─ YES → S6/S7 (적용 중인 쿠폰 기본 선택)
  │     │     ├─ "적용 중인 쿠폰" 섹션 + "적용 가능한 쿠폰" 섹션 각각 렌더링
  │     │     └─ 적용 가능한 쿠폰 없으면 해당 섹션 미노출
  │     │
  │     └─ NO → S1 (최대 혜택 쿠폰 자동 선택)
  │           └─ 정렬 우선순위 값이 가장 큰 쿠폰을 기본 선택
  │
  ├─ 사용 가능 쿠폰 0개 → S5 (empty view, CTA 미노출)
  │
  ├─ [유저 액션] 다른 쿠폰 탭 → 선택 변경 (S3) → 스낵바 (S4)
  ├─ [유저 액션] 만료 쿠폰 아코디언 펼침 → S2
  │
  └─ [유저 액션] CTA "쿠폰 적용하고 대출 찾기" 클릭
        ├─ POST /pcs/v1/coupon-info/{couponInfoId} 호출
        ├─ 기존 clickYn=true 쿠폰 모두 false로 초기화
        ├─ 새로 선택한 쿠폰만 true로 업데이트
        ├─ 성공 → LOAN_APPLY_CREDIT_INTRO 이동
        └─ 실패 → 현재 화면 유지 + 에러 토스트

```

## 6. 화면 정책 상세

### 6-1. 헤더

| 항목 | 값 |
|------|---|
| title | "내가 받은 쿠폰이에요" |
| sub (분기 1: clickYn=true 없음) | "최대 혜택 쿠폰을 자동으로 선택했어요" |
| sub (분기 2: clickYn=true 있음) | "지금 적용 중인 쿠폰이 있어요" |

### 6-2. 쿠폰 리스트 (ISSUED 쿠폰에 한함)

- 더보기/페이지네이션 없이 전체 리스트 노출
- 적용 가능한 쿠폰 섹션 타이틀 옆에 쿠폰 개수 표기
- **정렬 기준**: `benefitValue DESC → expireTime ASC → insertTime DESC`
- **최대혜택 뱃지**: 정렬 우선순위 값이 가장 큰 쿠폰에 표시

### 6-3. 쿠폰 선택 변경

- 사용 가능한 쿠폰(ISSUED) 2개 이상일 시, 다른 쿠폰 탭하면 선택 변경
- **쿠폰별 선택 해제 불가능** — 어떤 쿠폰도 선택하지 않은 상태는 존재할 수 없음
- CTA 항상 활성화 (사용 가능 쿠폰 1개 이상 보유 시)
- 쿠폰 만료 전이라면 대출 약정 후에도 적용 쿠폰 변경 가능

### 6-4. 하단 고정 CTA

| 항목 | 값 |
|------|---|
| 문구 | "쿠폰 적용하고 대출 찾기" |
| 랜딩 | `LOAN_APPLY_CREDIT_INTRO` |
| 활성 조건 | 사용 가능 쿠폰 1개 이상 |
| 비활성 조건 | 사용 가능 쿠폰 0개 → CTA 미노출 + empty view |

### 6-5. 기간 만료 쿠폰

- 기존 2탭 구조 제거, 만료 쿠폰은 하단 아코디언 배너로 이동
- ⌵ 클릭 시 EXPIRED 리스트 노출
- 정렬: 만료일 DESC (기존 유지)
- 더보기/페이지네이션 없이 전체 리스트 노출
- 만료된 쿠폰 없을 시 배너 영역 미노출
- **만료 쿠폰은 clickYn 업데이트, 최대혜택 뱃지, 혜택값 정렬 등 정책 적용 대상 아님**

### 6-6. 에러 정책

| 케이스 | 처리 | 유저 화면 |
|--------|------|---------|
| CTA 클릭 시 API 호출 실패 | 현재 화면 유지 | 에러 토스트: "일시적인 오류가 발생했어요. 잠시 후 다시 시도해주세요." |

## 8. API & 데이터

### CTA 클릭 시 호출

```
POST /pcs/v1/coupon-info/{couponInfoId}
```

- 선택된 couponInfoId 기준으로 `clickYn` 업데이트
- 기존 `clickYn=true` 쿠폰은 모두 `false`로 초기화 (토글 방식)
- 만료된 쿠폰은 적용 제외, ISSUED 쿠폰에 대해서만 적용

### BO 개선 요구사항

- `coupon_info`에 혜택값(benefitValue) 필드 추가
- 해당 값은 쿠폰 노출 및 기본 선택 순서를 결정하는 가중치로 사용
- BO 메뉴: FRM 메세지 > 쿠폰 관리 > 쿠폰 등록
- 필드명: "혜택 크기" (필수, 수정 가능)

## 10. UI 구조 (Figma 기반)

```
[Status Bar]
[Header]
  ├─ title: "내가 받은 쿠폰이에요"
  └─ sub: (분기별 상이)
[적용 중인 쿠폰] — clickYn=true 쿠폰 있을 때만 노출
  └─ [List/Basic] (115px) — 현재 적용 중인 쿠폰 카드
[적용 가능한 쿠폰 N] — 섹션 타이틀 + 쿠폰 수
  └─ [List/Basic] × N — 쿠폰 카드 리스트
      ├─ 아이콘 (40×40)
      ├─ 쿠폰명
      ├─ 혜택 내용
      ├─ 유효기간 + D-day
      └─ 최대혜택 뱃지 (최상단 쿠폰에만)

[Button/CTA] — 하단 고정 "쿠폰 적용하고 대출 찾기" (360×96)
[Component/Snackbar] — 쿠폰 변경 시 (321×52, CTA 위)
```
