# 차번호 입력 프로모션 (주유권 추첨 이벤트) — 통합 명세

## 1. 핵심 목표

- 프로모션을 통해 차량 번호 미입력 유저의 입력 행동을 유도하여 `is_own_car = 1` 유저 수 증가
- 퍼널 외부 입력 구조의 유효성 검증
- 핵심 KPI: 프로모션 시작 vs 종료 시점 `is_own_car = 1` 유저 수 순증
- 보조 KPI: 참여 완료 인원 7,022명 목표 / CRM CTR 0.64% / 배너 CTR 11.2% / 페이지 진입→입력 CVR 20.8%

## 2. 이벤트 스킴

| 항목 | 내용 |
|------|------|
| 대상자 | 핀다 전체 회원 (CRM 대상: 대고객) |
| 기간 | 2026.06.01 ~ 2026.06.30 |
| 유입구 | CRM(PUSH) + 스마트배너 |
| 컨셉 | 차 번호 입력 → 본인 명의 확인 → 주유권 실시간 추첨 |
| 리워드 ① | GS칼텍스 10,000원 주유권 × 50명 (0.75% 확률) |
| 리워드 ② | 랜덤 포인트 (미당첨자 전원) |
| 예산 | 총 68만 원 내외 (주유권 50만 + 포인트 18만) |

## 3. User Flow

```
[CRM 발송 / 스마트배너]
  │ 진입
  ▼
[프로모션 페이지] — 이름(자동매칭, 수정불가) + 차량번호 입력(유저 직접)
  │
  ▼ 특정 조건 만족 시 CTA 활성화 (별도 유효성 에러 메시지 없음)
[CTA 클릭: "실시간 주유권 추첨하기"]
  │
  ▼
◇ 프로모션 참여 여부 확인
  ├─ [참여 완료 유저] → 팝업 "이미 참여한 이벤트예요" → 확인 → 홈 이동
  └─ [미참여 유저] → 동의서 바텀시트 노출
        │
        ▼ 필수 동의 완료 + 확인 CTA 클릭
[로딩 페이지] (로띠 애니메이션)
  - 상단: 메인+서브 문안 (로띠)
  - 하단: 차담대 관련 안내 문안 (로띠)
  - 광고성 수신 동의 유저: 정보 수신 동의 날짜 토스트 노출
  │
  ▼
◇ 본인 명의 확인 (NICE 디앤알 API)
  ├─ [시스템 오류] → 시스템 오류 팝업 → CTA "다시 시도하기" → 프로모션 페이지
  ├─ [본인 명의 X] → 본인 명의 오류 팝업 → CTA "다시 입력하기" → 프로모션 페이지
  └─ [본인 명의 O] → 차량 정보 DB 저장 + is_own_car = 1 전환
        │
        ▼
  ◇ 주유권 추첨
    ├─ [당첨] → 주유권 당첨 결과 페이지
    └─ [미당첨] → 포인트 지급 결과 페이지
          │
          ▼ CTA 클릭
    [대출비교 인트로 페이지] — 차 번호 오토필
```

> **공통 네비게이션**: 모든 페이지 상단 좌측 `X` 표시 (← 아님). 클릭 시 핀다 메인홈으로 이동.

## 7. 기술 요구사항

- NICE 디앤알 API 활용 차량 정보 스크래핑 + 본인 명의 확인
- 차량 정보 기존 테이블에 저장
- 비교대출 단계에서 차량 정보 오토필
- 어트리뷰트 테이블 `is_own_car = 1` 전환
- 이름 자동입력 오류 시 백 로직으로 1회 재시도, 이후 실패 시 시스템 에러 팝업
- 프로모션 참여 여부는 리워드 지급 여부로 판단
