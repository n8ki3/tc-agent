# 비교대출 완료 추첨 포인트 지급 이벤트 (7월) — 통합 명세

> **기반 문서**: PRD (FQA-2245) + Figma + Tech Spec
> **작성일**: 2026-06-26
> **이해충돌 체크리스트 확정**: 2026-06-26
> **Epic**: [FQA-2259](https://findainc.atlassian.net/browse/FQA-2259)

---

## 1. 핵심 목표

- 비교대출 완료 유저 대상 추첨 이벤트로 대출 비교 완료율 및 재참여 유도
- 랜덤 포인트(100~1,000P) 즉시 지급 + 고액(10만P) 추첨 당첨 이원화

---

## 2. 이벤트 스킴

| 항목 | 내용 |
|------|------|
| 대상자 | 비교대출 완료 유저 |
| 기간 | 2026.07.01 ~ 2026.07.15 |
| 유입구 | CRM PUSH / 스마트배너 / 핀다소식탭 |
| 리워드 A | 핀다 포인트 100~1,000P 즉시 지급 (`rewardItemType=FINDA_POINT`) |
| 리워드 B | 10만P — 추후 수기 지급 (즉시 미지급) (`rewardItemType=VOUCHER`) |
| 중복 참여 | 전체 기간 1회 (CI 기준) |
| 집계 배너 | 인트로: "n명이 도전 중" (TC 제외) / 결과: "n명이 포인트를 받았어요" (TC 제외) |

## 3. User Flow

```
[CRM PUSH / 스마트배너 / 핀다소식탭]
  │ 유입
  ▼
[프로모션 인트로]
  │ 뒤로가기 → 이전 화면 이동
  │ [안내] 버튼 → 안내 바텀싯 → 닫기 → 프로모션 인트로 복귀
  │
◇ 중복 참여 여부
  ├─ [이미 참여 완료] → ※ [미정의: 화면 처리 정책 확인 필요]
  └─ [미참여] → CTA 선택
        │
        ▼
[본인인증] — 기존 한도조회 공용 화면 (TC 미생성)
        │
        ▼
[추첨 결과] — rewardItemType 기준 분기
  ├─ [FINDA_POINT] → 포인트 즉시 지급 결과 화면 (100~1,000P)
  │      └─ ※ [미정의: 화면 구성 Figma 확인 필요 — U3]
  │      └─ CTA 선택
  │            ├─ 가승인 유저 → 가승인 화면 이동
  │            └─ 올거절 유저 → 올거절 화면 이동
  └─ [VOUCHER] → 10만P 추후 수기 지급 안내 결과 화면
         └─ CTA 선택
               ├─ 가승인 유저 → 가승인 화면 이동
               └─ 올거절 유저 → 올거절 화면 이동
```

## 4. 화면 정책 상세

### 4.1 프로모션 인트로

| 항목 | 정책 |
|------|------|
| 뒤로가기 버튼 | 존재 — 선택 시 이전 화면으로 이동 |
| [안내] 버튼 | 선택 시 안내 바텀싯 노출 |
| 안내 바텀싯 닫기 | 프로모션 인트로로 복귀 |
| 집계 배너 | "n명이 도전 중" (TC 제외) |
| 이미 참여 완료 유저 재방문 | ※ [미정의 — U1: 정책 확인 필요] |

### 4.2 본인인증

| 항목 | 정책 |
|------|------|
| 화면 유형 | 기존 한도조회 공용 화면 (신규 화면 아님) |
| TC 생성 여부 | 미생성 (U6 결정) |

| 집계 배너 문구 | TC 제외 (U5 결정) |

## 5. 기술 요구사항

| 항목 | 내용 |
|------|------|
| 당첨 결과 분기 조건 | `rewardItemType` 필드 — `FINDA_POINT` vs `VOUCHER` |
