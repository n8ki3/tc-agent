# QA TC Generator — 웹 애플리케이션

## 개요

Kiro IDE 에이전트로 검증된 QA 분석 로직을 **독립 웹 서비스**로 확장하는 프로젝트입니다.
누구나 브라우저에서 PRD + Figma 링크를 입력하면 TC를 다운로드할 수 있는 것이 목표입니다.

---

## 서비스 플로우

```
┌─────────────────────────────────────────────────────────────┐
│  웹 브라우저                                                  │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  1. 입력                                            │    │
│  │     • PRD PDF 업로드 (드래그 앤 드롭)                │    │
│  │     • Figma 링크 입력                               │    │
│  │     • [분석 시작] 버튼                               │    │
│  └──────────────────────┬──────────────────────────────┘    │
│                         │                                   │
│                         ▼                                   │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  2. 분석 진행 (실시간 상태 표시)                      │    │
│  │     • PRD 분석 중... ✓                              │    │
│  │     • Figma 분석 중... ✓                            │    │
│  │     • 이해충돌 검토 중... ✓                          │    │
│  └──────────────────────┬──────────────────────────────┘    │
│                         │                                   │
│                         ▼                                   │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  3. 이해충돌 체크리스트 (인터랙티브)                   │    │
│  │     • 충돌 #1: [PRD 기준 ▼] 선택                    │    │
│  │     • 미정의 #2: [TC에서 제외 ▼] 선택               │    │
│  │     • [확정하고 TC 생성] 버튼                        │    │
│  └──────────────────────┬──────────────────────────────┘    │
│                         │                                   │
│                         ▼                                   │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  4. 결과 & 다운로드                                  │    │
│  │     • TC 미리보기 (HTML 렌더링)                      │    │
│  │     • [통합 명세 .md 다운로드]                        │    │
│  │     • [TC .md 다운로드]                              │    │
│  │     • [TC .html 다운로드]                            │    │
│  │     • [TC .xlsx 다운로드] (스프레드시트)              │    │
```

## 기술 스택

| 레이어 | 기술 | 역할 |
|--------|------|------|
| Frontend | Next.js + TailwindCSS | 입력 UI, 체크리스트, TC 미리보기 |
| Backend | Python (FastAPI) | 분석 엔진, LLM 오케스트레이션 |
| LLM | Claude API | PRD 분석, TC 생성 |
| Figma 연동 | Figma REST API | 디자인 파일 메타데이터 + 스크린샷 |
| PDF 파싱 | PyMuPDF | PRD PDF 텍스트/표/취소선 추출 |
| 스토리지 | 세션 기반 (인메모리) | 분석 상태 관리 |
| Export | openpyxl | xlsx 다운로드 |

## 개발 로드맵

| Phase | 상태 | 내용 |
|-------|------|------|
| **1. 프로토타입** | ✅ 완료 | Kiro IDE 에이전트로 핵심 로직 검증 |
| **2. 백엔드 API** | 🔲 예정 | 분석 엔진을 REST API로 분리 |
| **3. 프론트엔드** | 🔲 예정 | 웹 UI (입력 → 체크리스트 → 다운로드) |
| **4. 인프라** | 🔲 예정 | 배포 + 인증 + 파일 스토리지 |
| **5. 확장** | 🔲 예정 | Jira 연동, 회귀 TC 자동 식별, 팀 협업 |

## Phase 1 → Phase 2 전환 시 재사용 자산

## 핵심 기술 과제

| 과제 | 설명 | 접근 방향 |
|------|------|----------|
| PDF 파싱 | PRD PDF에서 텍스트/표/취소선 추출 | PyMuPDF + 구조화 파싱 |
| Figma API 연동 | MCP 없이 직접 REST API 호출 | Figma REST API + OAuth |
| LLM 오케스트레이션 | 다단계 분석을 안정적으로 실행 | 커스텀 파이프라인 (세션 상태 머신) |
| 체크리스트 상태 관리 | 비동기 사용자 입력 대기 | 세션 기반 상태 저장 + 폴링 |
| TC 품질 보장 | 근거 기반 원칙을 LLM이 준수하도록 | 출력 검증 레이어 + few-shot 예시 주입 |

## 차별점

| 기존 TC 자동 생성 도구 | 이 서비스 |
|----------------------|----------|
| PRD 텍스트만 입력 | PRD + Figma 디자인 동시 분석 |
| 바로 TC 생성 | 이해충돌 검토 → 사용자 확정 → TC 생성 |
| 추측 포함된 TC | 근거 기반 원칙 + 미정의 항목 분리 |
| 일회성 생성 | 명세 업데이트 시 TC 자동 동기화 |

## 현재 구현 상태

`webapp/` 폴더에 프론트엔드(Next.js) + 백엔드(FastAPI) 초기 구조가 세팅되어 있습니다.

```
webapp/
├── frontend/          ← Next.js + TailwindCSS
│   └── src/
│       ├── app/       ← 페이지 라우팅
│       └── components/
│           ├── InputStep.tsx       ← PRD 업로드 + Figma 링크 입력
│           ├── AnalyzingStep.tsx   ← 분석 진행 상태 표시
│           ├── ChecklistStep.tsx   ← 이해충돌 체크리스트
│           └── ResultStep.tsx      ← TC 미리보기 + 다운로드
├── backend/
│   └── app/
│       ├── routers/
│       │   ├── analyze.py    ← 분석 시작 API
│       │   ├── confirm.py    ← 체크리스트 확정 API
│       │   ├── generate.py   ← TC 생성 API
│       │   └── download.py   ← 파일 다운로드 API
│       └── services/
│           ├── pdf_service.py        ← PDF 파싱
│           ├── figma_service.py      ← Figma API 연동
│           ├── confluence_service.py ← Confluence API 연동
│           ├── llm_service.py        ← LLM 호출
│           └── export_service.py     ← md/html/xlsx 생성
└── README.md
```
