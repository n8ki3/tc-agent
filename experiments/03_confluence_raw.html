1. 목표(Objective)

  • 핵심 목표:

    • 프로모션을 통해 차량 번호 미입력 유저의 입력 행동을 유도하여 is_own_car = 1 유저 수를 증가시키고, 퍼널 외부 입력 구조의 유효성을 검증한다.

  • 서브 목표:

    • 확장된 CRM 풀을 기반으로 차담대 약정(신청·실행) 증가 가능성 확인

      • 자동차담보대출 한도조회 건수 - 전체 한도조회 건수 중 비중

        • 26.03 기준 30.24% → 10%p 상승 목표

    • Tech Spec. [26][2Q] 차번호 입력 프로모션 API HandOff

    • 향후 퍼널 외부 입력 구조 및 신규 유입 구조 설계를 위한 근거 확보

  • 핵심 KPI

    • 프로모션 시작 시점 vs 종료 시점 is_own_car = 1 유저 수 순증 (절대값)

  • 보조 KPI

    • 프로모션 기간 내 참여 완료 인원 수 (절대값)

      • 목표 참여 완료 인원: 7,022명

    • CRM/배너 CTR

      • 목표 CTR: CRM 약 0.64% / 배너 약 11.2%

    • 프로모션 페이지 진입 - 차량 번호 입력 CVR

      • 목표 CVR: 약 20.8%

  • 예상 비용, 성과 시뮬레이션 https://docs.google.com/spreadsheets/d/1Y4gmvAmHKV4gyqTCbn3QL6_cQWKGzYt82ln18UeDuK4/edit?gid=1634399393#gid=1634399393&range=B2:E3

    • 총 예산: 68만 원 내외

      • 주유권: 50만 원 / 포인트 지급 비용: 18만 원 내외

      • 자세한 사항은 시트 혹은 하단 포인트 지급 정책 참고

    • 주요 가정 및 예상 참여 인원 수 산정 배경

      • 실제로는 대고객이 대상자이지만 최근 30일 이내 로그인 한 유저(활성 유저) 기준으로 예상 인원 수와 목표 수치를 설정함.

      • 마수동 여부 / 차량 보유 여부에 따라 유저 그룹을 4개로 나눔.

      • 해당 프로모션은 차량 번호 입력 유도가 핵심 목적이라 소재 자체가 차량 관련 문구로 구성됨. 이에, 입력(행동) 경험 유무 에 따라 CTR(crm/배너 클릭률)이 다를 것으로 판단함.

      • 마수동 여부에 따라 노출 채널이, 기존 차량번호 입력 여부에 따라 참여 허들이 달라지므로 CVR(페이지 진입 - 차량번호 입력)도 그룹별로 다르게 적용함.

      • CTR 벤치마크는 고유가 지원금 프로모션(CRM 0.9%, 배너 13.4%)과 일반 평균(CRM 0.5%, 배너 10%)을 기준으로 했으며, CVR은 고유가 지원금(38.5%)과 마수동 유도 프로모션(28%)을 벤치마크로 활용함.
        단, 실제 성과 측정 시 그룹별 실측은 어려워 가중 평균으로 계산한 전체 통합 기준으로 관리할 예정.

2. 배경 (Background)

2.1. 배경

[26.04] 자동차 번호 입력 개선 제안

  • 정보 입력 단계에서 12.22% 가량이 퍼널을 이탈하고 있다.

  • 차량 정보 입력 여부에 따라 입력 유저가 전반적으로 더 적극적인 행동을 보이고 있다.

    • 자동차 정보 입력 여부에 따라 '대출결과 목록 → 상세 진입 전환율'이 약 33% vs 53%,

    • '상세 → 신청 전환율'은 21% vs 31%로 나타난다.

  • 퍼널 외부에서 정보를 사전에 입력받을 수 있는 경로가 없어, 유저가 혜택을 계기로 정보를 입력하는지에 대한 검증이 불가능하다.

2.2 기대효과

  • 단기적으로는 차번호 입력 프로모션을 통해 입력 전환율을 높이고, 사용자가 혜택을 계기로 실제로 정보를 입력하는지, 퍼널 외부 입력 경로가 유효한지 확인할 수 있다.

  • 이와 함께 부가적으로 퍼널 진입 전 정보 확보를 통해 사용자 경험 개선이 가능하다.

    • 특히 S3 유저와 같이 대출 니즈는 있으나 기존 상품에서 승인되지 않는 고객군에 대해 차량 정보를 기반으로 보다 적합한 상품을 제안할 수 있다. (참고: 세그먼트별 CRM 실험 아카이브)

    • 이를 통해 자담대 상품 등 CRM 풀 확대 및 추가 매출 기회 발굴이 가능하다.

  • 장기적으로는 해당 실험 결과를 바탕으로 퍼널 외부 입력 구조의 유효성을 검증하고, 퍼널 단축 및 새로운 입력 구조로의 확장 가능성을 확인할 수 있다.

  • 또한 프로필/정보입력 기반 서비스로의 확장 근거를 확보하고, 데이터 자산 축적 및 스코어링 고도화 등 금융 플랫폼으로서의 장기적인 경쟁력 강화로 이어질 것으로 기대한다.

3. 상세 기획 (Scheme)

3.1 이벤트 스킴(Scheme)

  • 대상자: 차번호 입력 X 유저

    • CRM 대상자: 대고객

대상자 조정 히스토리760

  • 기존에는 is_own_car = 0 OR NULL 유저 대상으로 차번호 입력 프로모션을 진행하려 했으나, 해당 값은 실제 차량 보유 여부를 정확히 의미하지 않아 타겟팅 한계가 존재함.

    • car=0
      → 실제 무차량 유저일 수도 있고, 차량이 있지만 한도조회 시 입력을 건너뛴 유저일 수도 있음

    • car=null
      → 한도조회 미진행 유저인지, 정보 만료(90일) 유저인지 구분 불가

    따라서 car=0/null만 대상으로 할 경우,
    실제 차량이 없는 유저 비중이 높아 프로모션 참여율 자체가 낮아질 가능성이 있음.

    이 경우:

    • 외부 입력 퍼널 자체의 유효성이 낮은 것인지

    • 단순히 차량 보유 유저가 적어서 반응이 낮은 것인지

    구분이 어려워 실험 결과 해석이 제한될 수 있음.

  • 모든 유저 대상으로 오픈하여, 차량이 있는 유저가 자발적으로 차번호를 입력하도록 유도.

  • 이를 통해:

    • 외부 입력 퍼널의 유효성 검증 가능

    • 혜택 기반 정보 입력 반응 확인 가능

    • 기존 차량 보유 유저의 차번호 입력 행동 강화 가능

  • 단, car=1 유저가 다수 포함될 수 있으므로 프로모션 직후 한도조회로 자연스럽게 연결되는 후속 액션 보완 필요.

  • 채널 및 유입구

    • CRM: PUSH

    • 스마트배너: 동일 모수의 프로모션 대상자에 노출 (마수동 여부 상관X)

  • 기간: 2026.06.01-2026.06.30

  • 컨셉: 주유권 쿠폰 지급 (이전 프로모션과 다른 성격의 리워드)

    • 프로모션 페이지에서 사용자로부터 자동차 번호 정보를 입력 받고, 본인 명의 확인을 통해 랜덤으로 주유권을 지급한다.

  • 리워드

    • GS칼텍스 주유권 1만 원 권 x 50명 지급

    • 주유권 미지급 시 랜덤 포인트 지급

이벤트 기간 중 1인당 1회만 참여할 수 있어요.
유효한 차 번호를 입력하고, ‘실시간 주유권 추첨하기’ 버튼을 눌러 본인 명의 인증이 완료된 고객님에 한해 이벤트 참여가 완료돼요.

이벤트 혜택 01
추첨을 통해 총 50명에게 GS칼텍스 10,000원 주유권 지급
10,000원 주유권 당첨 여부는 참여 직후 당첨 결과 페이지에서 바로 확인할 수 있어요.
당첨된 일자 기준 다음 월요일에 LMS(문자)로 주유권 쿠폰이 발송돼요.
주유권 사용기한은 발급일로부터 30일이에요.
사용 가능 주유소: 전국 GS칼텍스 주유소 및 충전소 (일부 사업장은 사용처에서 제외될 수 있으니, 사용 전 미리 확인하시기 바랍니다)
주유권 사용 조건
유효기간 내 1인 1일 1매만 사용할 수 있어요.
쿠폰 금액 초과 사용분에 한해서만 GS ALL 포인트 적립이 가능해요.
쿠폰의 현금 교환 및 타인 양도는 불가능해요.
동일 쿠폰 및 타 쿠폰과 중복 사용은 불가능해요.
쿠폰 금액 미만 결제 시 잔액 환불은 불가능해요.
셀프 주유기 사용이 어려운 경우 직원에게 문의 후 사용할 수 있어요.
중복 사용 및 중복 적립은 불가능해요.


이벤트 혜택 02
랜덤 포인트 지급

  • 주유권에 당첨되지 않은 고객님에게 랜덤 포인트가 지급돼요.

  • 랜덤 포인트는 핀다 포인트 형태로 제공돼요.

  • 지급 안내는 앱 내 팝업 또는 앱 푸시를 통해 발송돼요.

  • 지급된 포인트는 ‘핀다 앱 > 하단 핀다혜택 탭 > 포인트’에서 확인할 수 있어요.

  • 핀다 포인트는 제휴 모바일 상품권으로 교환하여 사용할 수 있어요.

  • 지급된 핀다 포인트는 지급일로부터 1년 후 자동 소멸돼요.

  • 주유권 당첨 고객님은 랜덤 포인트를 받을 수 없어요.

이벤트 대상자 및 혜택 제외자
다음에 해당하는 경우 이벤트 대상 및 혜택 지급 대상에서 제외될 수 있어요.

기타 안내
당첨 경품의 시가가 5만 원을 초과하는 경우, 관련 세법에 따라 제세공과금(원천징수세율 22%)이 부과되며 이는 당첨자 본인 부담이에요.
5만 원 이하 경품은 비과세 대상이에요.

6. 킥오프 싱크 관련

6-1. 디자인 킥오프 논의사항

  1. 동의서 관련
    동의서에 '광고성 정보 수신 동의'와 '마케팅 목적 수집·이용 동의' 두 항목이 현재는 모두 선택으로 구성되어 있는데, 광고성 수신 동의는 마케팅 수집·이용 동의가 선행되어야 하는 구조라 처리 방식 확인이 필요.

  • 두 항목을 독립적으로 각각 선택 가능하게 할지 여부

  • 마케팅 수집·이용 동의 없이 광고성 수신 동의만 단독 선택 허용 여부

  • 광고성 수신 동의 완료 토스트는 두 항목 모두 동의한 경우에만 노출할지 여부
    → 정보보안팀 확인 필요

  1. 프로모션 페이지 차량 번호 입력 처리 관련

  • 유효하지 않은 차량번호 입력 시 오류 메시지 노출 여부 관련

  • 값에 따라 입력 중간에 확인할 수 있는 게 있고, 입력 완료 시에 확인할 수 있는 게 있어서 테크 확인 필요

  • 7자리로 입력할 경우 버튼 활성화 & 이후 동의서 노출 전에 유효한 차량 번호 여부 검증하여 유효하지 않을 경우 팝업으로 오류 메시지 노출하는 방식 고려 필요

  1. 로딩 페이지 관련

  • 로딩 시간 기준 주피터팀 확인 필요 (최소/최대)

  • 텍스트 롤링 처리 방식 확인 필요 (프론트 처리 or 로띠 형식)

  1. 이탈 후 재진입 관련

  • 중간 이탈 후 재진입 시 처음부터인지, 이어서 진행 가능한지 (결과 페이지로의 랜딩) 확인 필요

  • 이탈할 경우에도 정보 저장되는지 확인 필요
