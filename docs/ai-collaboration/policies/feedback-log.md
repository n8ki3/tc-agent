## 패턴 분석

### 승격된 규칙 (3건 이상 또는 즉시 승격)

| 패턴 | 피드백 건수 | 승격 규칙 |
|------|-----------|----------|
| 물리적으로 다른 진입 경로를 하나로 합침 | 1건 (즉시 승격) | TC 분리 기준 — 진입 경로 |
| 선택/미선택 한쪽만 TC 작성 | 1건 (즉시 승격) | TC 분리 기준 — 선택 항목 양쪽 |
| 피그마에 존재하는 UI 요소인데 TC 미생성 | 3건 (뒤로가기 + 안내사항 + BO) | 공통 검증 패턴 섹션 신설 |

### 아직 규칙화 안 된 잠재 패턴

(피드백 누적 시 여기에 추가)

---

## 규칙 승격 기준

1. **즉시 승격** — 피드백 내용이 명확하고, 모든 피처에 범용 적용 가능한 경우
2. **3건 누적 후 승격** — 비슷한 유형의 피드백이 3건 이상 쌓이면 상위 규칙으로 일반화
3. **보류** — 특정 피처에만 해당하거나, 규칙화하면 과잉 TC 생성 우려가 있는 경우
