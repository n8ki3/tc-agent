# 기획문서(PRD) 분석 규칙

## 분석 대상

기획문서는 다음 형태로 제공될 수 있다:

- PDF export (위키/노션/컨플루언스)
- 마크다운 파일
- 채팅에 첨부된 문서
- **Confluence 페이지 URL** — API로 자동 수집

## Confluence 연동

- Confluence URL이 제공되면 `.env` 파일의 인증 정보를 사용하여 REST API로 페이지 내용을 가져온다.
- 필요한 환경변수: `CONFLUENCE_BASE_URL`, `CONFLUENCE_EMAIL`, `CONFLUENCE_TOKEN`
- API 호출: `GET {BASE_URL}/wiki/rest/api/content/{pageId}?expand=body.storage`
- HTML 응답을 파싱하여 텍스트로 변환 후 분석한다.
- `.env` 파일이 없거나 인증 실패 시 사용자에게 인증 정보를 요청한다.

## 추출해야 할 정보

기획문서에서 다음 항목들을 빠짐없이 추출한다:

1. **핵심 목표 / 성공 지표** — KPI, 현재 기준값, 목표값
2. **문제 정의** — 배경, 페인포인트, 근본 원인
3. **대상 사용자** — 핵심/부가 사용자 정의
4. **화면 정책** — 분기 조건, 노출/미노출 규칙, 텍스트 문구
5. **비즈니스 규칙** — 정렬 기준, 선택 로직, 상태 전이
6. **에러 정책** — 에러 케이스별 처리 방식, 유저 화면 메시지
7. **API 명세** — 엔드포인트, 호출 시점, 요청/응답 동작
8. **이벤트 로그** — 이벤트명, 파라미터, 발생 조건, 수집 중단 이벤트
9. **BO(백오피스) 요구사항** — 운영 도구 변경 사항
10. **미결 사항** — 아직 확정되지 않은 항목
11. **가드레일 지표** — 부작용 모니터링 기준

## 확정/미확정 판별 기준

| 표기 | 해석 |
|------|------|
| 일반 텍스트 | 확정된 정책 |
| ~~취소선~~ | 폐기된 안 (적용하지 않음) |
| "검토 중", "TBD", "확인 필요" | 미확정 → 미결 사항으로 분류 |
| ✅ 체크 표시 | 확정 완료 |
| 볼드/하이라이트 + "확정" | 최종 결정 사항 |
| "SCOPE OUT" | 현재 스코프에서 제외 |

## 사용자 피드백과 문서 충돌 시

- 사용자의 구두 피드백이 문서 내용과 다를 경우, **충돌된 부분을 명시적으로 알려주고 한 번 더 컨펌한다.**
- 예: "PRD 3-4에서 할인율형은 취소선 처리되어 폐기된 것으로 보이는데, 실제로 둘 다 사용하는 게 맞을까요?"
- 사용자가 컨펌하면 그때 반영하고, 명세에 "PRD와 상이 — 사용자 컨펌 기준" 주석을 남긴다.

## 버전 이력 처리

- 문서에 버전 이력이 있으면 **최신 버전 기준**으로 분석한다.
- 이전 버전에서 변경된 내용은 "변경 전 → 변경 후"로 추적하되, 최종 정책은 최신 버전을 따른다.
- 버전 간 충돌이 있으면 최신 날짜의 내용을 우선한다.

## 회의록 / 논의 내용 처리

- 기획문서 하단에 회의록이 포함된 경우, 회의에서 **확정된 결론**만 정책으로 반영한다.
- "논의 중", "추후 결정" 등은 미결 사항으로 분류한다.
- 회의록의 논의 과정(장단점 비교 등)은 참고 정보로만 취급하고, 정책으로 반영하지 않는다.
- ">> 결론 PRD 내에 반영" 같은 표기가 있으면 해당 결론이 확정된 것으로 판단한다.

## 다중 안(1안/2안) 처리

- 기획문서에 여러 안이 병기된 경우, **PRD 본문에 최종 반영된 안**만 정책으로 채택한다.
- ">> 결론" 또는 확정 표기가 없으면 미결 사항으로 분류한다.
- 채택되지 않은 안의 내용을 기대 결과에 포함하지 않는다.

## 출력 형식

기획문서 분석 결과는 `aidlc-docs/features/` 아래 통합 명세 파일에 반영하며, 다음 구조를 따른다:

1. 핵심 목표
2. 문제 정의 & 솔루션 요약
3. AS-IS → TO-BE 변경 요약
4. 화면 목록 (분기 조건 포함)
5. 서비스 프로세스 (User Flow)
6. 화면 정책 상세
7. API & 데이터
8. 이벤트 로그
9. 미결/확인 필요 사항
